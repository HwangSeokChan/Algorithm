### 문제에 대한 생각
- Dynamic programming
  - memorization
  - bottom-up
- 재귀함수 stack-overflow
  - 호출의 사이즈가 실행환경의 허용치가 넘어갈 경우 발생 ( 메모리 과부하 )
  - 문제 조건의 100000 개는 재귀함수 사용시 실행환경에 따라 오류처리 됨.
  - 고로, 재귀함수가 아닌 bottom-up 을 사용하여 실행하는 것이  
    복잡도 및 자원허용치면에서 안정적이다.
- 복잡도
  - normal : O(n^{2})
  - memorization : O(n)
  - bottom-up : O(n)
